一、因为DataTables调用的是pdfMake进行导出PDF，而pdfMake自带的字体无法支持中文，需要我们自己生成vfs_fonts.js进行替换，步骤如下：

1、安装Grunt及模块

	1.首先运行如下命令：
	npm install -g grunt-cli
	
	2.再到pdfMake源代码目录运行：
	npm install grunt --save-dev
	
	3.之后再安装需要用到的模块：
	npm install grunt-dump-dir --save-dev
	
	
2、编译自定义vfs_fonts.js文件
	1.将下载的ttf字体文件放到项目examples\fonts目录下，在pdfMake根目录下运行如下命令：
	grunt dump_dir
	
	2.如果运行成功将显示如下消息：
	File “build/vfs_fonts.js” created.
	Done, without errors.
	
	3.现在可以将build/vfs_fonts.js文件引到自己的项目中来，再进行注册自定义字体的设置。
	
	
3、在项目中注册自定义字体
	1.这里需要对pdfMake.fonts进行注册自己的自定义字体，在页面头部进行javascript载入初始化
	pdfMake.fonts  = {
	  msyh: {
		 normal: 'msyh.ttf'
			}
	};
	2.msyh是我自定义的字体名称，使用的是微软雅黑字体，在你的docDefinition中将默认字体设置成你前面向pdfMake注册过的字体就可以了。
	
	var docDefinition = {
  	 content: '测试pdfMake中文导出乱码', 
  	 defaultStyle: {
    		font: 'msyh'
  		}
	};
	
4、附件是我使用的pdfMake和DataTables还有我使用微软雅黑编译的vfs_fonts.js文件。

	pdfMake
	DataTables-1.10.8
	vfs_fonts.js
